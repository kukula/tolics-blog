# AGENTS.md

Guidance for AI agents working in this repo. Keep it accurate — update it when
conventions change.

## What this is

**Tolics Engineering Mind** — a personal Hugo blog (software architecture, AI,
neurodivergent-friendly systems). Static site, custom **freerange** theme,
deployed to **GitHub Pages** via GitHub Actions (`.github/workflows/deploy.yml`,
`hugo --minify` on push to `main`). A second workflow auto-posts new entries to
Bluesky.

## Repo layout

- `content/posts/*.md` — posts, **flat files** with YAML front matter (no leaf
  bundles). `content/about.md`, `content/search.md` — standalone pages.
- `themes/freerange/` — the theme. CSS is one file:
  `themes/freerange/static/css/main.css`. Templates in
  `themes/freerange/layouts/` (`_default/single.html`, `_default/list.html`,
  `index.html`, `partials/`).
- `assets/images/covers/<ContentBaseName>.webp` — per-post cover masters
  (Hugo-processed; **not** `static/`).
- `scripts/` — shell utilities (`*.sh`): cover generation, link checks, date
  sync, em-dash fixing.
- `hugo.toml` — site config (`pagination.pagerSize = 12`, menus, OG defaults).
- `STYLE_GUIDE.md` — **writing** voice/style (read before drafting prose).
- `ILLUSTRATIONS_PLAN.md`, `scripts/ILLUSTRATION_STYLE.md` — cover pipeline + art
  direction.

## Common commands

```bash
hugo server -D                       # local dev preview at :1313
hugo --quiet --destination public    # one-off build
hugo new posts/my-post.md            # scaffold a post
./scripts/check_external_links.sh    # validate links
```

For **visual** verification of HTML/CSS changes, use the `preview-blog` skill
(deterministic build → serve → headless screenshot). For styling work, use the
`freerange-frontend` skill (design tokens, neobrutalist patterns, cover system).

## Conventions

- **Posts** are flat `.md` with front matter: `title, date, lastmod, draft,
  author, description, tags, categories`, optional `pinned`, `series`,
  `illustration`. Match an existing recent post's shape. Follow `STYLE_GUIDE.md`
  for prose.
- **Design:** neobrutalist — bold solid borders, hard offset shadows (no blur),
  teal accent, oklch colors, CSS variables (never hardcode tokens). Everything
  must work in **both** light and dark (`data-theme`). **Minimal functional UI:
  if an element isn't needed, remove it** (no decorative frames, no redundant
  buttons, no hover motion for its own sake). **DRY UI: a shared element
  (pagination, post card, button) is implemented once — one partial or one
  class — and reused, never copied between templates** (duplication is how
  styles silently drift).
- **Covers:** one illustration per post by filename convention; rendered via
  `partials/cover.html`; generated by `scripts/generate_illustrations.sh`
  (Google "Nano Banana" → WebP master). Don't hand-edit covers — regenerate.
- **Hugo:** image processing only works from `assets/` (`resources.Get` +
  `.Resize`/`.Process`); `static/` is copied verbatim. Built HTML is **minified**
  (grep `class=foo`, not `class="foo"`). Paths key on `.File.ContentBaseName`,
  not `slug`.

## Don't

- Don't commit **secrets**: `env` (holds `GEMINI_API_KEY`) is gitignored — keep
  it that way.
- Don't commit `public/`, `resources/_gen/`, or `.claude/settings.local.json`
  (all gitignored). `.claude/skills/` **is** tracked and shared.
- Don't commit unrelated root-level draft `.md` files (e.g.
  `change-your-story.md`) — they are scratch, not posts.
- Commit/push only when asked. Branch off `main` for non-trivial work unless told
  otherwise.

## Git

Recent history commits directly to `main` (solo blog). End commit messages with
the project's co-author trailer when committing as an agent.
