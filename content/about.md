## I'm Tolic

Why systems fail interests me more than the features they ship. Always has.

That instinct built a career. 17 years of production SaaS. An MSc in Applied Mathematics. Observability infrastructure, database migrations, distributed systems across three continents. Melbourne now. Building agentic AI systems with the Claude API. Writing about what AI-native software actually looks like.

## Currently Building

**[Promin Dev](https://promin.dev)**
Code loses its *why* over time. Requirements rot. Tests outlive their intent. Nobody remembers which constraint forced that weird workaround. Promin is a knowledge graph that reconnects requirements, code, tests, and analytics. A living map of intent, for engineers and AI agents both. Built on the Claude API, the Claude Agent SDK, KuzuDB, and Elixir/OTP.

**[Evolvr](https://evolvr.tech)**
CommOps for Melbourne property managers. Automates email triage, response drafting, invoice validation, and reporting. Retrieval-augmented generation (RAG) knowledge base. Human-in-the-loop approval gates. A [PII substitution layer](/posts/2026-02-10-ai-knows-your-clients-phone-number/) keeps client data out of LLM context.

## Writing

Posts that define my professional vision:

- **[Hybrid vs AI-Native: Why Developers Became Middle Management](/posts/2026-03-05-hybrid-vs-ai-native/)**. In hybrid orgs, developers relay intent between humans and AI. AI-native orgs cut out the relay.
- **[Code Is a Lossy Format](/posts/2026-01-19-code-is-a-lossy-format/)**. Source code discards intent. LLMs expose the problem. Graph-based models fix it.
- **[Beyond the Horseless Carriage](/posts/2026-02-05-beyond-the-horseless-carriage/)**. We build AI that mimics human workflows. AI-native software should look like a living organism.
- **[Beyond the Chat Window: UX on Steroids with LLMs](/posts/2026-02-28-beyond-the-chat-window/)**. Chat is the laziest AI integration. Components that remember users and act on context are the real opportunity.
- **[Context Engineering: The Skill That Actually Matters](/posts/2026-03-16-context-engineering/)**. Not prompt engineering. Deciding what goes into the context window, and why.
